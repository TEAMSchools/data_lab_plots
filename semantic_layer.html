The Problem

Data Without a Semantic Layer Is Broken

Every team builds their own queries, their own definitions, and their own dashboards — and they rarely agree.

🔀

Inconsistent Metrics

Attendance means something different in every report. One team counts days, another counts events. Leadership gets three different numbers for the same question.

⏳

Data Team as Bottleneck

Every question requires a ticket, a SQL query, and a two-week wait. Non-technical staff can't access the data they need to do their jobs.

🏝️

Siloed Systems

SIS, LMS, HRIS, assessments — each system lives in its own world. Combining them requires deep tribal knowledge that lives in one person's head.

🔓

No Access Control

Either everyone can see everything (a FERPA nightmare) or nobody can see anything. There's no middle ground for role-based, row-level data access.

🤖

AI Has No Foundation

LLMs are powerful, but asking an AI to query raw data is asking it to guess your definitions. Without governed semantics, AI answers are unreliable.

🔄

Duplicated Work

The same join logic, the same business rules, and the same filters get written over and over across dozens of reports by dozens of people.

The Solution

A Semantic Layer Fixes All of This

A semantic layer sits between your raw data warehouse and every tool that consumes data — defining business logic once, in one place, for everyone.

Capabilities

What Our Semantic Layer Can Do

Built on Cube, our semantic layer provides a full stack of data capabilities — from governed access to AI-powered interpretation.

📐

Governed Metrics

Every measure and dimension is defined in code — versioned, reviewed, and consistent across every tool that consumes it.

🔒

Row-Level Security

User identity and group membership are embedded in signed JWT tokens. Cube enforces data access policies automatically — no manual filtering needed.

⚡

Pre-Aggregation & Caching

Common queries are pre-computed and cached. Complex aggregations that used to take minutes return in milliseconds.

🤖

AI-Ready Data

Our AI layer (Google Gemini 2.5) queries governed views — not raw tables — so answers are accurate, contextualized, and FERPA-compliant.

🔌

Universal API

One REST API works for all consumers: custom apps, BI tools, notebooks, and AI agents. Build once, connect anything.

🧩

Self-Service Exploration

Non-technical staff can filter, pivot, and visualize data in purpose-built apps — no SQL, no tickets, no waiting.

What querying the semantic layer looks like

Instead of writing SQL joins across five tables, any consumer — app, AI, or analyst — calls one structured API. The semantic layer handles the rest.

# Ask: how many students are enrolled per grade level this year? # Zero SQL. Zero joins. Zero guessing at table names.
rows = client.query( measures = ["Students.count"], dimensions = ["Students.gradeLevel"], filters = [ CubeFilter("Students.schoolYear", "equals", ["2025-2026"]), CubeFilter("Students.enrollmentStatus", "equals", ["Active"]), ], order = {"Students.gradeLevel": "asc"}, )
# Returns clean, typed rows — ready for charts, tables, or AI interpretation # [{"Students.gradeLevel": "K", "Students.count": 145}, # {"Students.gradeLevel": "1", "Students.count": 152}, ...]

Technology

Built on a Modern,
Open Stack

Every component is production-grade, open-source-friendly, and designed to grow with the organization — not lock you into a vendor.

  • ⚡
    Cube.dev — Semantic Layer OLAP engine that defines measures, dimensions, and views. Pre-aggregates queries and enforces access policies.
  • 🐍
    Python + Streamlit — App Layer Rapid web app development. A new dashboard can be live in hours by pointing at any existing Cube view.
  • 🤖
    Google Gemini 2.5 Flash — AI Layer Fast, cost-effective LLM for data interpretation. Queries only governed views — no raw table access.
  • 🔐
    Google OAuth 2.0 + JWT — Auth Layer Single sign-on with existing Google Workspace accounts. Row-level security embedded in signed tokens.
  • 📊
    Plotly — Visualization Layer Interactive, publication-quality charts rendered directly in the browser.
